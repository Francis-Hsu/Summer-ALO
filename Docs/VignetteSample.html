Introduction

alocv is a package that implements the method proposed by Wang et al. (2018). In short, it approximates the leave-one-out cross-validation (LOOCV) risk for several standard models, using only the full-data solution. Since the model no longer needs to be fitted nn times, the LOOCV estimates can be obtained very efficiently, even in the case when n is so large that leave-one-out is infeasible.

This package currently supports LOOCV approximation for Elastic-net GLM (glmnet) and SVM (e1071).

Usage with glmnet

We begin by loading our package, of course:

#> Loading required package: glmnet
#> Loading required package: Matrix
#> Loading required package: foreach
#> Loaded glmnet 2.0-16
glmnetALO(X, y, glm_obj, alpha, standardize, type.measure)

For demonstration, we will now create some artifical data of size n=500n=500 and p=200p=200, with k=100k=100 non-zero coefficients:

n = 500
p = 200
k = 100
beta = rnorm(p, 0, 1)
beta[-(1:k)] = 0

X = matrix(rnorm(n * p, 0, sqrt(1 / k)), n, p)
y = X %*% beta + rnorm(n, 0, 0.5)
y[y >= 0] = 2 * sqrt(y[y >= 0])
y[y < 0] = -2 * sqrt(-y[y < 0])

For such a data, even the glmnet with its state-of-art implementation needs some time to do the LOOCV:

ptm = proc.time()
CV_el = cv.glmnet(X, y, alpha = 0.5, nfolds = n, 
                  grouped = F, intercept = T, standardize = T, type.measure = "mse")
proc.time() - ptm
#>    user  system elapsed 
#>   13.25    0.40   13.07
ptm = proc.time()
GLM_el = glmnet(X, y, lambda = CV_el$lambda, alpha = 0.5, intercept = T, standardize = T)
ALO_el = glmnetALO(X, y, GLM_el, 0.5, standardize = T, type.measure = "mse")
proc.time() - ptm
#>    user  system elapsed 
#>    0.34    0.10    0.17
plot(ALO_el)
